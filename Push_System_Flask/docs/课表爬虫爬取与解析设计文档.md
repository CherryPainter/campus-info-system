# 课表爬虫：爬取与解析设计文档

> 适用范围：Push_System 课程表模块 —— 重庆工程学院 eams 教务系统（`http://jwxt.cqie.cn:8081/eams/`）
> 文档版本：v1.0 ｜ 整理时间：2026-07-07
> 维护人：后端开发

---

## 1. 文档目的

本文档说明课程表"爬取 → 解析 → 入库 → 前端展示"的完整链路，定位每一层的关键代码与数据格式，并记录**已修复的解析陷阱**与**尚未攻克的已知风险（重点是"切换学期来爬"）**。

读完后应能：
- 知道改爬取逻辑改哪里、改解析逻辑改哪里、改入库/展示改哪里；
- 清楚教师字段是怎么来的（**运行时从教务页解析，绝不写死**）；
- 明白"切换学期来爬"目前处于什么状态、有哪些真实风险。

## 2. 系统概述

| 项 | 说明 |
|----|------|
| 目标站点 | 重庆工程学院 eams 教务系统（需登录 + 验证码） |
| 技术栈 | Python + Playwright（异步）+ BeautifulSoup + pytesseract（验证码 OCR） |
| 爬取粒度 | **整学期**：选择"教学周=全部"后，一次拿到本学期全部周次的课表 |
| 数据源 | 两条路径互补：① 渲染后的 HTML 表格；② 页面内嵌的 `TaskActivity` JS（含整学期周次位图 + 教师） |
| 运行入口 | `app/cqie-course-timetable/main.py`（必须在 `app/cqie-course-timetable/` 目录下运行） |
| 后端存储 | MySQL（`courses` 表，软删除 `is_deleted`） |

## 4. 整体数据流

```
登录(eams) → 验证码OCR → 选"教学周=全部" → 等待整学期渲染
        │
        ├─[路径A] 解析渲染表格 <table>  → _extract_schedule（处理 rowspan=2/4 大课）
        │
        └─[路径B] 解析页面 JS 的 TaskActivity → _parse_activities（整学期周次位图 + 教师）
                    │
                    ▼
        CourseProcessor.run()  ── 输入 grid / 已存 HTML
          ├─ _split_cell_courses      拆分同一单元格内多门课
          ├─ _extract_course_info     正则提取 课程名/代码/周次/教室
          ├─ _parse_weeks_string      周次串 → list[int]
          ├─ _merge_teacher_from_map  按课程代码补全教师（仅补空值，来源=源HTML）
          └─ merge_consecutive_courses 同门课跨连续节次合并为一条(periods 列表)
                    │
                    ▼
        processed_course_table.json → 批量入库(软删旧+插入新)
                    │
                    ▼
        MySQL courses 表 → 前端按 week_day/period_idx/weeks 渲染 + is_course_in_week 周过滤
```

## 5. 爬取层（main.py）

`CourseTableTool` 类核心方法：

| 方法 | 职责 |
|------|------|
| `_login` | Playwright 填账号密码、处理登录错误重试（最多 3 次） |
| `_download_and_recognize_captcha` | 下载验证码图片 → pytesseract OCR |
| `_select_all_weeks` | 在"选择教学周"下拉框选"全部"，触发整学期渲染 |
| `_parse_activities` | 解析页面 `new TaskActivity(...)` JS，得到**整学期**课程（周次位图 + 教师 + 教室） |
| `_build_grid_from_activities` | 把 TaskActivity 展开成 `unit_count=12`（每天 12 节）的网格 |
| `_extract_schedule` | 解析渲染后的 `<table>`，处理 `rowspan`（2/4 节大课） |
| `_weeks_bitmap_to_str` / `_expand_weeks_str` | 周次位图 ↔ 紧凑区间串（如 `[2-7,10-15,17]`） |
| `get_course_table` | 入口：爬取当前视图课表 |
| `get_all_weeks_course_table` | 入口：自动/交互模式爬取整学期 |

**CLI 参数**（来自 `main()`）：

```bash
python main.py                  # 爬取当前视图（默认单周）
python main.py --all-weeks      # 自动爬取全部周次（1-20 周，约 30 分钟）
python main.py --all-weeks --interactive --max-weeks 18   # 交互模式，手动切周
```

> ⚠️ 注意：**当前 CLI 没有 `--semester` / `--semester-id` 参数**。爬虫只能爬"教务系统当前默认学期"。详见第 9 章。

## 6. 解析层（course_processing/process_course_data.py）

`CourseProcessor` 是规范解析器，接收 `grid`（来自路径 A 或已存 HTML）后产出 `final_courses`。

### 6.1 单元格多课拆分 `_split_cell_courses`
eams 同一节次不同周次的课会**直接拼接在同一单元格**，无分隔符，例如：
```
劳动教育与实践 4(121210060.15) ;([2-7]周);体育 4(120810460.38) ;([10-15] [17]周,教学区足球场1(校本部))
```
按 `(数字.数字)` 课程代码的位置切片，拆成多门独立课。

### 6.2 正则解析 `_extract_course_info`
单门课文本格式：
```
课程名(课程代码)([周次]周,教室(校本部))
```
例：`体育 4(120810460.38) ;([10-15] [17]周,教学区足球场1(校本部))`

### 6.3 周次解析 `_parse_weeks_string`
将 `"[2-7]周"`、`"[10-15] [17]周"` 等转为 `list[int]`：`[2,3,4,5,6,7,10,11,12,13,14,15,17]`。
支持去括号、去"单/双/周"等噪声字符。

### 6.4 跨节合并 `merge_consecutive_courses`
同一门课在相邻节次（如第 1-2 节、第 3-4 节）的记录，合并为一条，`periods=[1,2,3,4]`，`period_idx` 取首节。供前端渲染成"跨节高卡片"。

## 7. 教师字段来源（重点：绝不写死）

**问题背景**：eams 渲染出的课表 `<table>` 单元格里**不含教师姓名**，所以只解析渲染表格时教师必然为空。

**正确来源**：教师数据就在课表页的 JS 里 —— 每个 `TaskActivity` 前都有 `var actTeachers=[{name:"罗强"}]`。

**实现**（`process_course_data.py`）：
- `extract_teacher_map_from_html(html)`：解析源 HTML 的 `var actTeachers`，得到 `{课程代码: 教师名}` 映射。**完全运行时从教务页实时提取，零硬编码。**
- `CourseProcessor.__init__(teacher_map=...)` 接收该映射；
- `_merge_teacher_from_map()`：仅给"自身为空、且课程代码命中映射"的课程补教师；**映射里没有的，保持空字符串，绝不臆造。**

> 全代码库不存在写死的"教师姓名/代码对照表"。生产爬取链路（`main.py` 的 `_extract_schedule(teacher_mapping=...)`）本就会把教师插进单元格；离线重导入脚本 `reimport_with_teacher.py` 也复用同一套从源解析的逻辑。

## 8. 入库层与前端展示

### 8.1 入库
- `CourseRepository.create_batch` 批量插入；
- 导入前对旧数据做**软删除**（`is_deleted=1` + `deleted_at` + `deleted_reason`），不物理删除；
- `weeks` 字段以 **JSON 数组字符串**存储，例如 `"[2,3,4,5,6,7,10,11,12,13,14,15,17]"`（注意：入库后经 ORM 可能变成双重编码 `"\"[19]\""`，周过滤需兼容）。

### 8.2 前端合并卡片 `Course.tsx → mergeAdjacentCourses`
同门课跨多节合并为一张高卡片的条件是：课程名相同 **且** 教师/教室匹配 **且** 节次严格相邻。
因教师字段历史上部分为空，合并条件中加入了 `fieldsMatch()` 空值兼容：**任一端为空（null/undefined/''）即视为匹配通过**，避免"同门课被拆成多个卡片"。非空时仍要求严格相等。

### 8.3 周过滤 `course_routes.py → is_course_in_week`
根据当前教学周，判断一条课程是否在该周上课。兼容 `weeks` 字段的双重编码：先剥外层引号再 `json.loads`，失败则退回 `parse_weeks` 文本解析。

## 9. ⚠️ 已知风险与现状：切换学期来爬（当前最大难关）

> 本章**如实**记录现状与风险，不回避、不粉饰。下一步要攻克的就是它。

### 9.1 现状
- `main.py` **只爬"当前默认学期"（eams 登录后默认 = id `271` = `2025-2026-2`）+ "全部"教学周**。
- 代码与 CLI **均无学期切换能力**（无 `accessSemester` 调用、无 `--semester-id` 参数）。
- 已保存的 `output/course-data/raw/course_meta.json` 显示：可选学期共 **28 个**（id 从 `291`=2026-2027-1 到 `9`），当前学期 `271`。
- 之前记忆里写的"已实现学期切换"**不准确** —— 实际未实现，仅确认了 eams 前端存在该接口。

### 9.2 eams 学期切换机制（来自已存页面 JS）
切换学期完全靠前端 Ajax（非我们代码调用）：
```javascript
// course_table.html 内
$.ajax({
  url: "/eams/accessSemester!access.action",
  data: { pagePath: "/courseTableForStd.action", "semester.id": $(obj).val() }
});
// 学期列表来自：/eams/dataQuery.action?dataType=semester
```
即：先拿学期列表 → 选中目标 `semester.id` → 调 `accessSemester!access.action` → 页面重新渲染该学期课表。

### 9.3 已知风险清单（未验证项）
1. **登录态保持**：切学期的 Ajax 是否要求已登录 session？切换后 cookie/session 是否失效？未验证。
2. **"全部"视图是否仍生效**：切到新学期后，选"教学周=全部"是否仍渲染整学期？还是会回落到某一周？未验证。
3. **TaskActivity 数据是否更新**：切学期后，页面内嵌 `TaskActivity` 是否携带新学期限的周次位图+教师？未验证。
4. **周数上限**：各学期周数不同（如 16/18 周），`_get_all_weeks_auto(max_weeks=20)` 上限需按学期动态，否则漏周。
5. **解析格式跨学期一致性**：课程代码/名称/教室格式跨学期是否一致？现有正则假设可能失效。
6. **教师映射跨学期**：`var actTeachers` 在新学期页面是否仍出现、格式是否一致？未验证。
7. **多学期并存会互相覆盖**：当前入库逻辑是"全量软删旧数据 + 插入新数据"，**没有 `semester_id` 隔离**。若爬两个学期，后爬的会把前爬的软删掉。

### 9.4 改造方向建议（仅分析，非已承诺实现）
- 在 `CourseTableTool` 增加 `switch_semester(page, semester_id)`：调用 `accessSemester!access.action` 并等待重渲染；
- CLI 增加 `--semester-id`；
- `courses` 表增加 `semester_id` 列，入库按 `(semester_id, ...)` 隔离，周过滤/前端透传 `semester_id`；
- 上述 7 条风险需**真机逐步验证**，不能凭推断。

## 10. 已修复的关键解析陷阱（历史记录，供避坑）

| # | 现象 | 根因 | 修复 |
|---|------|------|------|
| 1 | "体育 4" 被截断成 "4" | `_split_cell_courses` 回溯找课名时遇空格即停 | 从停止字符集移除空格/制表符 |
| 2 | 同门课跨节被拆成多个卡片 | 前端 `mergeAdjacentCourses` 要求 `teacher===teacher` 严格相等，部分教师为空导致失败 | 加 `fieldsMatch()` 空值兼容 |
| 3 | 课程少且全在下午 | `_extract_schedule` 行映射整体 +1 错位（表头行未跳过） | 改为从 `period_name`（第一节/第一、二节）反算 `period_idx`/`periods` |
| 4 | 按周过滤失效（如第19周混进专业课） | `is_course_in_week` 不认 DB 双重编码 `"[19]"` | 加剥引号 + `json.loads` 兼容 |
| 5 | 教师字段全空 | eams 渲染表格不含教师，只解析表格自然为空 | 新增 `extract_teacher_map_from_html` 从源 JS 提取（非写死），`_merge_teacher_from_map` 仅补空值 |

## 11. 运行与维护

### 11.1 依赖与环境
- Python 3.12+，需 `playwright`、`beautifulsoup4`、`pytesseract`（另装 tesseract OCR 引擎）；
- 账号密码在根目录 `.env`（`JWXT_USERNAME` / `JWXT_PASSWORD`），无头模式由 `JWXT_HEADLESS` 控制（默认 `true`）；
- **必须在 `app/cqie-course-timetable/` 目录下运行** `python main.py`。

### 11.2 离线重处理（不重新爬取）
当只改解析逻辑、手头已有 `output/course-data/raw/course_table.html` 时：
```bash
cd app/cqie-course-timetable
python reimport_with_teacher.py     # 重新解析 + 教师补全 + 软删旧 + 全量重导入
```

### 11.3 产物位置
- 原始 HTML：`output/course-data/raw/course_table.html`
- 处理后 JSON：`output/course-data/processed/processed_course_table.json`
- 运行日志：`output/logs/`
- 以上 `output/` 已被 `.gitignore` 忽略，**不进入版本库**。
