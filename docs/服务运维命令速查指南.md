# 服务运维命令速查指南

## 一、服务管理

### 启动/停止/重启

```bash
# 开发模式运行
cd /opt/Push_System_Flask
source venv/bin/activate
python3 run.py

# 后台运行（生产环境建议用 systemd）
nohup python3 run.py > logs/run.log 2>&1 &

# systemd 服务管理
sudo systemctl start push-system    # 启动
sudo systemctl stop push-system     # 停止
sudo systemctl restart push-system  # 重启
sudo systemctl status push-system   # 查看状态
sudo systemctl enable push-system   # 开机自启
sudo systemctl disable push-system  # 关闭自启
```

### 查看实时日志

```bash
# 应用日志
tail -f /opt/Push_System_Flask/logs/app.log

# systemd 日志
sudo journalctl -u push-system -f

# systemd 标准输出/错误
tail -f /opt/Push_System_Flask/logs/service.out
tail -f /opt/Push_System_Flask/logs/service.err
```

## 二、数据库管理

### 状态检查

```bash
# 查看数据库状态（表清单、行数、缺失表/字段）
python3 init_db.py status
```

### 初始化/迁移

```bash
# 首次部署：完整初始化（建表 + 种子数据）
python3 init_db.py init

# 增量迁移（补建缺失的表/列/索引）
python3 init_db.py migrate

# 仅写入种子数据（admin 用户、模块配置）
python3 init_db.py seed

# 重置数据库（删除所有表 → 重建 → 种子数据，需确认）
python3 init_db.py reset
```

### 数据库备份

```bash
# MySQL 备份
mysqldump -u <用户名> -p <数据库名> > backup_$(date +%Y%m%d).sql

# 恢复
mysql -u <用户名> -p <数据库名> < backup_YYYYMMDD.sql
```

## 三、部署与更新

### 首次部署

```bash
# 安装系统依赖
sudo apt update && sudo apt install -y python3 python3-pip python3-venv tesseract-ocr tesseract-ocr-chi-sim

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 安装 Playwright
playwright install chromium
playwright install-deps chromium

# 创建目录
mkdir -p data/auth data/electricity/charts data/weather logs output
```

### 更新代码

```bash
cd /opt/Push_System_Flask
git pull origin main

# 更新 Python 依赖
source venv/bin/activate
pip install -r requirements.txt

# 更新前端
cd admin-frontend
npm install
npm run build
cd ..

# 迁移数据库（如有变更）
python3 init_db.py migrate

# 重启服务
sudo systemctl restart push-system
```

## 四、环境配置

### 修改配置

```bash
# 编辑 .env 文件
nano /opt/Push_System_Flask/.env

# 生成安全密钥
openssl rand -hex 32  # SECRET_KEY
openssl rand -hex 16  # ADMIN_TOKEN

# 验证配置（启动时自动加载）
python3 run.py
```

### 关键环境变量

| 变量               | 说明              | 命令生成               |
| ------------------ | ----------------- | ---------------------- |
| `SECRET_KEY`       | JWT 签名密钥      | `openssl rand -hex 32` |
| `ADMIN_TOKEN`      | 管理令牌/初始密码 | `openssl rand -hex 16` |
| `JWXT_USERNAME`    | 教务系统用户名    | 手动填写               |
| `JWXT_PASSWORD`    | 教务系统密码      | 手动填写               |
| `WECOM_WEBHOOK`    | 企业微信 Webhook  | 手动填写               |
| `QWEATHER_API_KEY` | 和风天气 API Key  | 手动填写               |

## 五、常见问题排查

### 端口占用

```bash
# 检查端口占用
sudo lsof -i :29528
# 或
sudo ss -tlnp | grep 29528

# 杀死占用进程
sudo kill -9 <PID>
```

### 权限问题

```bash
# 设置目录权限
sudo chown -R www-data:www-data /opt/Push_System_Flask/data
sudo chown -R www-data:www-data /opt/Push_System_Flask/logs
sudo chmod -R 755 /opt/Push_System_Flask/data
```

### Tesseract OCR 问题

```bash
# 检查安装
which tesseract
tesseract --version

# 检查中文语言包
tesseract --list-langs | grep chi_sim

# 安装中文包
sudo apt install -y tesseract-ocr-chi-sim
```

### Playwright 问题

```bash
# 安装浏览器依赖
playwright install-deps chromium

# 检查浏览器
playwright install chromium

# 设置内存优化（低内存服务器）
export CHROMIUM_FLAGS="--disable-dev-shm-usage --no-sandbox"
```

### 时区问题

```bash
# 查看当前时区
timedatectl

# 设置上海时区
sudo timedatectl set-timezone Asia/Shanghai
```

### 9.6 紧急处理流程

```bash
# 1. 服务异常 → 查看日志
sudo journalctl -u push-system -f

# 2. 服务无法启动 → 检查端口占用
sudo ss -tlnp | grep 29528

# 3. 数据库连接失败 → 检查数据库服务
sudo systemctl status mysql
mysql -u push_user -p push_system

# 4. 磁盘空间不足 → 清理日志
find /opt/Push_System_Flask/logs/ -name "*.log.*" -mtime +7 -delete

# 5. 内存溢出 → 临时重启服务
sudo systemctl restart push-system
```
