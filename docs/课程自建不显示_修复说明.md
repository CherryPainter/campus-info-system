# 课程模块"自建课程不显示" 修复说明

## 现象
手动「新增课程 / 导入」建出来的课，在课程模块默认视图里完全看不到；从教务爬来的课正常显示。

## 根因（学期错配）
- **界面默认按当前学期过滤**：`course_meta.json` 的 `current_semester_name = 2025-2026-2` → DB 学期 id **20252**，列表/课表查询都按 `semester_id == 20252` 过滤。
- **手动建课按"今天日期"推导学期**：创建分支用 `derive_current_semester()`，今天（7 月）被算成春季学期 → **20251**。
- 结果：自建课程被打上 `20251`，而界面只看 `20252` → 被整体过滤掉，完全不可见。
- 注：之前（日志 #21）只修了"去重键加 semester_id"和"course_meta 缺失时列表 fallback"，**没改创建路径本身**，所以错配依旧。本次是从根上把两端对齐到同一来源。

## 改动清单
**后端 `Push_System_Flask/app/api/course_routes.py`**（需重启后端）
1. `create_or_update_course` 创建分支：学期归属改为「前端当前查看学期 → 回退 course_meta 当前学期 → 最后按日期」，与界面过滤一致。
2. 编辑模式新增节次：补全 `semester_id/semester_name/academic_year/term`（顺带修了编辑新增节次漏填学期可能入库报错的隐患）。
3. `/import` 端点：导入按钮也按 course_meta 当前学期打签。

**前端 `admin-frontend/src/pages/Course.tsx` + `src/api/course.ts`**（需重新 build 部署 dist）
1. `handleSave` 建课时把 `selectedSemester` 传给后端（挂到当前查看学期）。
2. 去掉「上课周次」默认只填当前单周：原逻辑让课程只在该单周显示、切周即消失；现改为留空 = 所有周都显示。
3. `course.ts` 的 `Course` 接口补 `semester_id?`。

**数据修复脚本 `Push_System_Flask/repair_course_semester.py`**（用 ORM，幂等）
- 将 `semester_id=20251 且 course_code 以 CRAWL- 开头` 的 **16 门自建课**改挂 `20252`（立即可见）。
- 排查确认：`semester_id=0` 的 33 门均为已软删除（本就不显示）；`20251` 其余 64 门是真实春季教务课，保留（切到 20251 学期可见）。

## 生效步骤
1. 重启后端（加载新 `course_routes.py`）。
2. 前端 `npm run build` 并重新部署 dist。
3. 已运行的修复脚本把历史 16 门错配自建课归位；之后手动建课即与当前学期一致、立即可见。

## 验证
- 后端 `py_compile` 通过；前端改动最小化、类型安全。
- 修复前统计：20252=405（可见）、20251=80（含 16 自建 + 64 真实春季）、semester_id=0=33（均软删除）。
