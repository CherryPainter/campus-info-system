## IP黑名单管理

### 功能说明
IP黑名单系统提供以下功能：
- 自动检测并封禁恶意IP（基于请求频率和安全事件）
- 手动管理IP黑名单（添加/移除/启用/禁用）
- 安全事件记录和查询
- 过期记录自动清理

### 数据库表
- `ip_blacklist` - IP黑名单记录
- `ip_security_events` - 安全事件日志

### API端点

#### 1. 获取黑名单列表
```bash
GET /api/admin/ip-blacklist
Authorization: Bearer <JWT_TOKEN>

# 参数
page=1                # 页码
per_page=20           # 每页数量
only_active=true      # 只显示启用的记录
```

#### 2. 添加IP到黑名单
```bash
POST /api/admin/ip-blacklist
Authorization: Bearer <JWT_TOKEN>
Content-Type: application/json

{
    "ip_address": "192.168.1.100",
    "reason": "恶意访问",
    "duration_hours": 24,
    "note": "测试封禁"
}
```

#### 3. 移除IP from黑名单
```bash
DELETE /api/admin/ip-blacklist/192.168.1.100
Authorization: Bearer <JWT_TOKEN>
```

#### 4. 启用/禁用黑名单记录
```bash
PUT /api/admin/ip-blacklist/192.168.1.100/toggle
Authorization: Bearer <JWT_TOKEN>
Content-Type: application/json

{
    "active": false
}
```

#### 5. 获取安全事件
```bash
GET /api/admin/ip-blacklist/events
Authorization: Bearer <JWT_TOKEN>

# 参数
page=1
per_page=20
event_type=rate_limit    # 可选过滤
severity=high            # 可选过滤
```

#### 6. 清理过期记录
```bash
POST /api/admin/ip-blacklist/cleanup
Authorization: Bearer <JWT_TOKEN>
```

### 自动封禁阈值

系统在 `app/services/ip_blacklist_service.py` 中定义了自动封禁阈值：

```python
AUTO_BLOCK_THRESHOLDS = {

        'window': 60,         # 在60秒内
    },
    'security_violation': {
        'violations': 5,      # 5次安全违规
        'window': 300,        # 在300秒内
    },
}
```

## CSRF防护

### 功能说明
CSRF（Cross-Site Request Forgery）防护模块为Session认证提供安全防护。

**注意**: JWT认证不需要CSRF防护，因为JWT不依赖Cookie。

### 工作原理
1. 用户登录时生成CSRF Token，存储在Session中
2. 前端在状态更改请求（POST/PUT/DELETE/PATCH）中携带CSRF Token
3. 后端验证请求中的Token与Session中的Token是否匹配

### CSRF Token传递方式
1. **请求头**: `X-CSRF-Token: <token>`
2. **表单字段**: `_csrf_token: <token>`
3. **JSON字段**: `_csrf_token: <token>`

### 获取CSRF Token
```bash
GET /api/auth/csrf-token
```

响应示例：
```json
{
    "status": "success",
    "data": {
        "csrf_token": "abc123..."
    }
}
```

### 前端集成示例

#### JavaScript (Fetch API)
```javascript
// 1. 获取CSRF Token
fetch('/api/auth/csrf-token')
    .then(res => res.json())
    .then(data => {
        const csrfToken = data.data.csrf_token;
        
        // 2. 在POST请求中携带Token
        fetch('/api/some-endpoint', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'X-CSRF-Token': csrfToken
            },
            body: JSON.stringify({...})
        });
    });
```

#### JavaScript (Axios)
```javascript
// 添加请求拦截器
axios.interceptors.request.use(config => {
    const csrfToken = getCsrfTokenFromCookie();
    if (csrfToken) {
        config.headers['X-CSRF-Token'] = csrfToken;
    }
    return config;
});
```

### 豁免路径
以下路径不需要CSRF防护（在 `csrf_protect.py` 中配置）：
- `/api/auth/login` - 登录接口
- `/api/auth/refresh` - 刷新Token
- `/api/auth/logout` - 登出接口
- `/api/health` - 健康检查

### 安全建议
1. 所有状态更改请求（POST/PUT/DELETE/PATCH）必须使用Session认证 + CSRF Token
2. API接口继续使用JWT认证（无CSRF风险）
3. 定期轮换CSRF Token（例如在用户权限变更时）

## Session管理

### 功能说明
系统支持JWT + Session双认证机制：
- **JWT认证**: 用于API访问（无CSRF风险）
- **Session认证**: 用于Web访问（需要CSRF防护）

### 数据库表
- `server_sessions` - 服务端Session存储

### Session模型字段
```python
class ServerSession(Base):
    session_id      # Session ID (UUID)
    user_id         # 用户ID
    ip_address      # 客户端IP
    user_agent      # 浏览器User-Agent
    created_at      # 创建时间
    expires_at      # 过期时间
    last_activity   # 最后活动时间
    is_active       # 是否激活
```

### API端点

#### 1. 获取当前用户的活跃Session
```bash
GET /api/auth/sessions
Authorization: Bearer <JWT_TOKEN>
```

#### 2. 撤销指定Session
```bash
DELETE /api/auth/sessions/<session_id>
Authorization: Bearer <JWT_TOKEN>
```

#### 3. 撤销所有Session
```bash
DELETE /api/auth/sessions
Authorization: Bearer <JWT_TOKEN>
```

### Session配置
在 `app/__init__.py` 中配置Session：

```python
# Session过期时间（默认7天）
SESSION_EXPIRE_DAYS = 7

# Session配置
app.config['SESSION_COOKIE_SECURE'] = True      # HTTPS only
app.config['SESSION_COOKIE_HTTPONLY'] = True    # 防止XSS读取
app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'   # 防止CSRF
```

### 安全建议
1. 设置合理的Session过期时间
2. 启用HTTPS（生产环境）
3. 使用HttpOnly Cookie（防止XSS）
4. 使用SameSite Cookie（防止CSRF）
5. 定期清理过期Session

## 文件上传安全

### 使用文件上传安全模块

```python
from app.utils.file_upload_security import validate_file_upload, FileUploadError

@app.route('/upload', methods=['POST'])
def upload_file():
    try:
        # 验证文件
        file = validate_file_upload(
            request.files['file'],
            allowed_extensions=['.jpg', '.png', '.pdf'],
            max_size=5 * 1024 * 1024  # 5MB
        )
        
        # 处理文件（file已通过验证）
        filename = file['secure_filename']
        # ... 保存文件
        
        return jsonify({'status': 'success'})
    except FileUploadError as e:
        return jsonify({'status': 'error', 'message': str(e)}), 400
```

### 配置选项

在 `app/utils/file_upload_security.py` 中：

```python
# 最大文件大小（默认5MB）
MAX_FILE_SIZE = 5 * 1024 * 1024

# 允许的文件类型
ALLOWED_MIME_TYPES = {
    'image/jpeg': ['.jpg', '.jpeg'],
    'image/png': ['.png'],
    'application/pdf': ['.pdf'],
}

# Magic Bytes签名
FILE_SIGNATURES = {
    'image/jpeg': [b'\xff\xd8\xff'],
    'image/png': [b'\x89PNG\r\n\x1a\n'],
}
```

## 故障排查

### IP黑名单功能不工作
1. 检查 `ip_blacklist` 表是否创建
2. 检查安全中间件是否正确集成（`app/utils/security.py`）
3. 查看日志：`grep "IP黑名单" logs/app.log`

### CSRF Token验证失败
1. 确认使用Session认证（JWT不需要CSRF）
2. 确认前端正确传递CSRF Token
3. 确认Session未过期

### Session管理问题
1. 检查 `server_sessions` 表是否创建
2. 检查Session过期时间配置
3. 查看Session清理日志
