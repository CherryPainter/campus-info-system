## 三、登录 IP 封禁（v6.8.2 修复）

- 全局 `before_request` 将 `/api/auth/login`、`/api/auth/login_mfa` 列入白名单（security.py:417-420），入口处**跳过**全局 IP 黑名单（仅记审计），避免误封自己 IP 后无法进后台。
- 路由内部**显式** `_ip_is_blocked` 检查：被封 + 凭据错误 → 403 拒绝访问（日志记 `blocked`，不再写"密码错误"）；被封 + 凭据正确 → 管理员自助解封通道（凭正确凭据可登录后在后台解封自己）。

## 四、账号锁定与登录信号感知

- **账号锁定已彻底移除（v6.10.1）**：源码全局无 lock_account 逻辑（grep 确认），admin 不再会因爆破被自锁。
- 登录失败信号感知为五维滑动窗口（5 分钟），全部只限流/临时封，**绝不锁账号**：

| 维度 | 触发阈值 | 处置 | 源码位置 |
|---|---|---|---|
| 同 IP 同账号失败 | ≥5 | 限流该 IP 30min | ip_blacklist_service.py:57 |
| IP 跨账号（不同账号数） | ≥3 / ≥5 | 限流5min / 临时封1h | :68 |
| IP 枚举（不同不存在用户名数） | ≥8 | 限流10min | :88 |
| IP 总量失败 | ≥30 | 限流2min | :99 |
| 账号遭多 IP 围攻（不同来源 IP 数） | ≥5 | 临时封攻击源 1h | :111 |

## 五、IP 黑名单与自动封禁（Layer B）

- `AUTO_BLOCK_THRESHOLDS`（v6.10.2 起自动封禁改为按 `block_duration_hours` **限时**，可自动恢复；仅手动封禁可永久）：

| 类型 | 窗口/阈值 | 封禁时长 | 源码位置 |
|---|---|---|---|
| rate_limit（疑似 DDoS/扫描） | 60s / 100 次 | 24h | :23 |
| security_violation（SQLi/XSS 等） | 300s / 5 次 | 72h | :29 |
| large_request（大请求） | 60s / 10 次 | 12h | :34 |
| file_upload_abuse（上传滥用） | 300s / 20 次 | 48h | :39 |

- 边缘攻击检测（before_request 内 `_run_security_checks`，security.py:380-408）：
  - 敏感路径（.env/.git 等）→ 403 + 记 `suspicious_path`
  - 可疑请求（路径遍历/空字节）→ 400 + 记 `path_traversal`
  - SQL 注入 / XSS 扫描 → 400 + 记 `sql_injection` / `xss`
- 自动封禁触发后入 `ip_security_events` 表；手动封禁可在后台永久/限时。

## 六、限流（Flask-Limiter）

- 全局默认（extensions.py:33）：`60 per minute`、`10 per second`、`500 per hour`；存储 `storage_uri=os.getenv("REDIS_URL") or "memory://"`（v6.11.0 起：配置 REDIS_URL 即接入 Redis，未配置回退内存）。
- 路由级（extensions.py:45-48）：`strict=10/min`（登录/login_mfa/敏感认证）、`moderate=30/min`（一般 API）、`lenient=100/min`（公开查询）。
- 状态：v6.11.0 已修复。limiter 已接入 Redis（`REDIS_URL`），未配置时回退内存；多 worker/重启不再丢失限流状态。

## 十、审计与可观测

- 每个请求记 `AUDIT_REQUEST`（IP / METHOD / PATH / UA），security.py:447。
- 安全事件入 `ip_security_events` 表；安全配置变更写日志。

## 审计建议汇总（按风险排序）

1. **SECRET_KEY 生产未固定**：重启即全部 Token/Session 失效，且重启窗口可被重放。务必在 `.env` 写死强随机值。（v6.11.0 已修复：生产环境 `DEBUG=false` 时 `SECRET_KEY` 缺失直接启动失败）
2. **MFA 未全局强制**：建议对 admin 强制开启 TOTP。（v6.11.0 已修复：`FORCE_ADMIN_MFA` 默认开启，强制管理员 MFA）
3. **Flask-Limiter 限流存储**：v6.11.0 已接入 Redis（`REDIS_URL`），未配置回退内存；多 worker/重启不再丢失限流状态。（原审计建议已落地）
4. **CORS 默认含 localhost:5173/5174**：生产应缩到真实前端域名。
5. 历史永久封禁记录（如 1.202.112.53）不会被 v6.10.2 追溯改限时，需后台手动处理。
